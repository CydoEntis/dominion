<remaining_work>

Next feature (user confirmed they want this):

**Resume on startup** — settings checkbox `resumeOnStartup: boolean`
- When enabled, on app launch each restored session re-launches its original agent command
- After re-launching, automatically sends `/resume` + Enter to the terminal
- Should be a checkbox in Settings under a "Sessions" section
- The session's `agentCommand` and `cwd` are already persisted in layout.json (PersistedSession shape has agentCommand, cwd, conversationId)
- The `/resume` text should only be sent if the session had an agentCommand (not plain shell sessions)

Implementation scope:
1. Add `resumeOnStartup: z.boolean().default(false)` to AppSettingsSchema in ipc-types.ts
2. Add checkbox to SettingsForm.tsx under Sessions section
3. In useLayoutRestore.ts (renderer), after a session is created on restore, if resumeOnStartup is true and session has agentCommand, write `/resume\r` to the session via SESSION_WRITE IPC after a short delay (let the shell initialize)
</remaining_work>
